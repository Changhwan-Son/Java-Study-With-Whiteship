## 프리미티브 타입 종류와 값의 번위 그리고 기본 값
> 프리미티브 타입 = 기본 타입

정수 타입에는 byte, char, short, int, long 이 있고 실수 타입에는 float, double 이 있고 그리고 논리 타입에는 boolean 이 있다.

## 프리미티브 타입과 레퍼런스 타입
> 레퍼런스 타입 = 참조 타입

### 레퍼런스 타입

프리미티브 타입이 리터럴을 직접 저장하는 타입이었다면, 레퍼런스 타입은 객체(Object)의 번지를 참조하는 타입으로 배열, 클래스, 열거, 인터페이스 타입을 말한다. 

프리미티브 타입은 실제 값을 변수에 저장하지만, 레퍼런스 타입은 변수에 번지(주소값)을 저장해 주소값을 통해 참조한다. 주소값을 참조하여 사용한다하여 참조 타입이라고 부른다. 

메모리상으로 보면 변수가 스택 영역에 생성되고 객체는 힙 영역에 생성되어 힙 영역에서의 주소값을 스택 영역에 저장하고 참조하여 사용한다. 

주소값을 갖고 있기 때문에 ==, != 연산 시 주소값을 비교하게된다. 즉, 동일한 객체를 참조하는지를 알아볼 때 사용된다. 

### 메모리 영역 
> 이전 1주차에서 빼먹은 내용

JVM은 운영체제에서 할당받은 메모리 영역을 다음과 같은 세부 영역으로 구분하여 사용한다. 

<strong>메소드(Method) 영역 </strong>

메소드 영역에는 코드에서 사용되는 클래스 파일들을 클래스 로더를 통해 읽어들여 각 클래스의 필드(field) 데이터, 메소드(method) 데이터, 메소드 코드, 생성자 코드 등을 분류해서 저장한다. JVM이 시작할 때 생성되고 모든 스레드가 메소드 영역을 공유한다. 

<strong>힙(Heap) 영역</strong>

힙 영역은 객체나 배열이 생성되는 영역이다. JVM 스택 영역에서 생성된 변수가 이 객체나 배열을 참조한다. 참조하는 변수가 없다면 의미가 없는 객체가 된다. JVM은 이 의미 없는 객체를 가비지 컬렉터(Garbage Collector)를 이용해 알아서 제거한다. 따라서 개발자는 객체 제거를 따로 해줄 필요가 없다. 사실 따로 하는 방법도 딱히 없다. 

<strong>JVM 스택(Stack) 영역 </strong>

JVM 스택 영역은 각 스레드마다 하나씩 존재하며, 각 메소드가 실행될 때마다 프레임이라는 게 생성되어 거기에 로컬 변수 스택이있다. 로컬 변수 스택에 프리미티브 타입 변수나 레퍼런스 타입 변수가 저장된다. 프리미티브 타입 변수는 스택에 직접 값을 가지고 있지만, 레퍼런스 타입 변수는 스택에 힙 영역이나 메소드 영역에 있는 객체의 주소를 가지고 있다. 


### String  

문자열을 저장하는 String도 클래스이다. 문자열을 String 변수에 바로 저장하는 것이 아니라, String 객체에 문자열이 저장되고 String 변수는 Stinrg 객체를 참조한다. 따라서 String 변수에는 String 객체의 주소값이 저장된다. 

자바는 문자열 리터럴이 동일하다면 String 객체를 공유하도록 되어있다. 
```
String str1 = "문자열";
String str2 = "문자열";
```
이렇게 같은 문자열을 저장하면 str1과 str2은 동일한 String 객체를 참조하게되어 str1 == str2 가 true 가 된다. new 연산자를 이용하여 직접 새로운 객체를 만들게되면 이 비교연산자는 달라진다.

## 리터럴
```
int score = 90;
```

위 코드에서의 90처럼 소스코드 내에서 직접 입력된 값을 리터럴(literal)이라고 부른다. 

리터럴은 값을 종류에 따라 정수 리터럴, 실수 리터럴, 문자 리터럴, 논리 리터럴로 구분된다. 

사실 리터럴은 상수(constant)와 거의 비슷한 의미지만, 프로그램에서 상수는 "값을 한 번 지정하면 변경할 수 없는 변수"의 의미가 있기 때문에 이와 구분해서 리터럴이라고 한다. 

<정수 리터럴>
- 소수점이 없는 정수 리터럴은 10진수로 간주한다.
- 0으로 시작하는 정수 리터럴은 8진수이다.
- 0x로 시작하고 숫자 혹은 A~F로 구성된 리터럴은 16진수이다. 

<소수 리터럴>
- 소수점이 있는 리터럴은 10진수 실수로 간주한다. 
- 대문자 혹은 소문자 e가 있는 리터럴은 10진수 지수와 가수로 간주한다. ex ) 5E2 = 5 x 10 ^ 2

<문자 리터럴>
- 작은따옴표로 묶은 텍스트는 문자 리터럴로 간주한다. 

<문자열 리터럴>
- 큰따옴표로 묶은 텍스트는 문자열 리터럴로 간주한다.큰따옴표 안이 비어있어도 문자열 리터럴로 간주된다.

<논리 리터럴>
- true나 false는 논리 리터럴로 간주된다.

## 변수 선언 및 초기화하는 방법

### 변수의 선언 
형식 : 타입 변수이름;
```
int age;
double value;
int x, y, z;
```
변수 이름은 메모리 주소에 붙여진 이름이다. 프로그램은 변수 이름을 통해서 메모리에 접근하고, 그곳에 값을 저장하거나 그곳에 있는 값을 읽는다. 

변수 이름에는 자바 언어에서 정해진 명명 규칙이 있다.
- 첫 글자는 문자이거나 '$', '_' 이어야 하고 숫자로 시작할 수 없다. 
- 영어 대소문자가 구분된다. 
- 첫 문자는 소문자로 시작하고, 다른 단어가 붙으면 첫 문자는 대문자로 붙인다. 
- 문자 수는 상관없다. 
- 예약어는 사용 불가. 

### 초기화 

선언된 변수에 값을 저장하는 것을 초기화 라고 한다. 변수는 가능하면 선언과 동시에 적절한 값으로 초기화하는 것이 바람직하다. 멤버 변수는 초기화하지 않아도 변수의 타입에 맞게 기본값으로 초기화가 이루어지지만 지역변수는 사용하기 전에 반드시 초기화가 이루어져야한다. 

초기화는 대입 연산자를 통해 이루어진다. 
```
int num = 1;    // 선언과 동시에 초기화

int value;      // 선언한 다음 초기화
value = 2;
```

## 변수의 스코프와 라이프타임

변수는 중괄호 {} 블록 내에서 선언되고 사용된다. 중괄호를 사용하는 곳은 클래스, 생성자, 메소드 등이 있다. 

기본적으로 변수는 "선언된 블록에서만 사용이 가능하다"는 사용 규칙이 있다. 

메소드 블록 안에서 선언된 변수를 지역 변수(local variable)이라고 한다. 지역 변수는 메소드 실행이 끝나면 자동으로 없어진다. 

클래스 블록 안에서 선언된 변수를 멤버 변수(member variable)이라고 한다. 멤버변수는 클래스 변수와 인스턴스 변수로 나뉜다. 모든 객체가 공통적으로 똑같은 속성을 가질 때 사용하는 것이 클래스 변수, 객체별로 다른 속성을 가질 때 사용하는 것이 인스턴스 변수이다. 클래스 변수는 클래스가 메모리에 올라갈 때 생성(메모리 할당)되며, 인스턴스 변수는 객체가 생성될 때 생성된다. 
```
class GameCharacter
    static int height = 200;        // 클래스 변수, 모든 객체에 적용
    String name;                    // 인스턴스 변수, 값이 고정되지 않아서 각 객체에 사용 가능 
```

> 다른 언어에서의 전역 변수가 자바에서는 클래스 변수, 인스턴스 변수로 사용되는 것 같다. 클래스 변수는 다른 클래스에서도 사용이 가능하지만, 인스턴스 변수는 같은 클래스에서만 사용이 가능하다.
